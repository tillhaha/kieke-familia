Week planner addition

A "Generate Shopping List" button added to the /week page header. Clicking it POSTs to /api/shopping/generate for the current week, then navigates to /shopping.

API routes (/api/shopping/)

GET  /items            → all unchecked items, grouped by category
POST /items            → add item; upsert memory
PATCH /items/[id]     → update category (inline reassign)
DELETE /items/[id]    → delete (also used for check-off)
POST /generate         → aggregate week ingredients, filter blacklist & dupes, append
GET  /categories       → list (with seed on first call)
POST /categories       → create
PATCH /categories/[id] → rename / reorder
DELETE /categories/[id] → delete (items move to uncategorized)
GET  /blacklist         → list terms
POST /blacklist         → add term
DELETE /blacklist/[id]   → remove term

Generate logic

1. Fetch all DayPlan rows for the current week where lunchMealId or dinnerMealId is set
2. Load the linked Meal records and collect all ingredients[]
3. Lowercase + trim each ingredient; check against ShoppingBlacklist (substring match)
4. Check against existing ShoppingItem names (exact match after normalize) — skip dupes
5. Look up ShoppingItemMemory for each surviving ingredient to pre-assign category
6. Bulk-insert new items
