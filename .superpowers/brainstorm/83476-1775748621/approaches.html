Batch Import — which approach?

All three reuse the existing /api/meals/parse + /api/meals endpoints. Pick the flow that fits your needs.

A

Textarea + progress bar

One big textarea. URLs go one-per-line. Full recipe texts are separated by ---. Hit "Import All" — items parse and save sequentially with a live counter. Done.

Pros

  • Fewest clicks
  • Easy to paste a bulk list of URLs
  • No new page needed

Cons

  • No review before saving
  • Errors require re-running failures
B

Parse → review queue

Paste URLs / recipe texts (same format as A), then click "Parse All". Items parse in the background and appear as review cards showing name + type + diet. Accept or skip each one, then "Save accepted".

Pros

  • Catch parse mistakes before saving
  • Skip duplicates or bad results

Cons

  • Extra step before anything is saved
  • More UI to build
C

Multi-slot (add rows)

A list of rows — each row has a text input (URL or short text) with an expand button for long recipe pastes. Add/remove rows freely. Parse individually or all at once.

Pros

  • Fine-grained control per item
  • Easier to mix URLs and text

Cons

  • Most work to build
  • More clicks for large batches
