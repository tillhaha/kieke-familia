API routes (/api/shopping/)

GET  /items            → all unchecked items, grouped by category
POST /items            → add item; upsert memory
PATCH /items/[id]     → update category
DELETE /items/[id]    → delete (also used for check-off)
POST /generate         → push ingredients from a given week, filter blacklist & dupes, append
GET  /categories       → list (seeds defaults on first call)
POST /categories       → create
PATCH /categories/[id] → rename / reorder
DELETE /categories/[id] → delete (items → uncategorized)
GET  /blacklist         → list terms
POST /blacklist         → add term
DELETE /blacklist/[id]   → remove term
