Design — Data Model

4 new Prisma models, all scoped to familyId

ShoppingCategory
id       String (cuid)
familyId  String (FK → Family)
name      String
order     Int

@@unique([familyId, name])
ShoppingItem
id         String (cuid)
familyId   String (FK → Family)
name       String
quantity   String? (optional)
categoryId String? (FK → Category)
checked    Boolean (default false)
createdAt  DateTime

// checked items deleted on GET, not stored long-term
ShoppingItemMemory
id        String (cuid)
familyId  String (FK → Family)
itemName  String (lowercased)
categoryId String (FK → Category)

@@unique([familyId, itemName])
ShoppingBlacklist
id        String (cuid)
familyId  String (FK → Family)
term      String (lowercased)

@@unique([familyId, term])

Default categories (seeded on first visit)

Fruit/Vegetables · Meat · Fish · Milk Products · Bread · Condiments · Sauces · Drinks · Frozen · Nuts/Snacks

Created for the family the first time the shopping page loads. Fully editable/deletable.

Checked items strategy

When an item is checked off, it is deleted from the DB immediately (optimistic UI hides it first). This keeps the list clean and avoids accumulating stale data. No "done" section.
